API Documentation | Nataraj AI

Introduction

This document provides an overview of the available API routes, their methods, and the expected input and output formats for Nataraj AI. If you were intended to use the application, please head to https://nataraj-ai.web.app.

1. Home

Route: /

Method: GET

Description: Returns a simple HTML page with all the documentation of API.

Response: HTML content of the homepage.

2. Get Blogs

Route: /blogs

Method: GET

Description: Returns a JSON response containing the content of all .md files in the static/blogs directory.

Response: A JSON array of blog content.


Example Response:
[
    {
        "title": "Blog 1",
        "content": "Content of blog 1"
    },
    {
        "title": "Blog 2",
        "content": "Content of blog 2"
    }
]

3. Create User Data

Route: /createuserdata/{uid}

Method: GET

Description: Creates user data if it doesn't exist in Firestore.

URL Parameters: uid (string) - The UID of the user.

Response: A JSON object indicating success or failure.


Example Response:
{
    "success": true,
    "message": "Data Added Successfully!"
}

4. Contact Us

Route: /contactus

Method: POST

Description: Accepts contact form data (email and text) and stores it in Firestore.

Request Body: A JSON object containing email and text.


Example Request Body:
{
    "email": "user@example.com",
    "text": "This is a contact message."
}

Response: A JSON object indicating success.


Example Response:
{
    "success": true
}

5. Fetch Projects

Route: /fetchprojects/{uid}

Method: GET

Description: Fetches project IDs associated with a user ID from Firestore.

URL Parameters: uid (string) - The UID of the user.

Response: A JSON object containing project IDs.


Example Response:
{
    "success": true,
    "projects": [{project1}, {project2}]
}

6. Change Project Visibility

Route: /changeprojectvisibility/{pid}/{visibility}

Method: GET

Description: Changes the visibility of a project in the Firebase RTDB.

URL Parameters: pid (string) - The project ID.

URL Parameters: visibility (string) - The new visibility status.

Response: A JSON object indicating success or failure.


Example Response:
{
    "success": true
}

7. Delete Project

Route: /deleteproject/{pid}

Method: GET

Description: Deletes a project from the Firebase RTDB.

URL Parameters: pid (string) - The project ID.

Response: A JSON object indicating success or failure.


Example Response:
{
    "success": true
}

8. Delete Account

Route: /deleteaccount/{uid}

Method: GET

Description: Deletes a user account from Firebase Auth and Firestore.

URL Parameters: uid (string) - The UID of the user.

Response: A JSON object indicating success or failure.


Example Response:
{
    "message": "Account deleted successfully"
}

9. Create Project

Route: /createproject

Method: POST

Description: Creates a new project with an uploaded file and stores the data in Firebase RTDB and Firestore.

Request Body: Multipart form data containing the file and project details.

Response: A JSON object indicating success and the new project ID.


Example Response:
{
    "success": true,
    "projectID": "nataraj-12345"
}

10. Update Dance Form

Route: /updatedanceform

Method: POST

Description: Updates the dance form for a project in the Firebase RTDB.

Request Body: A JSON object containing the project ID and dance form.


Example Request Body:
{
    "pid": "nataraj-12345",
    "dance_form": "Freestyle, Hip Hop"
}

Response: A JSON object indicating success.


Example Response:
{
    "success": true
}

11. Generate Dance

Route: /generatedance

Method: POST

Description: Generates dance steps for a project using the Gemini API and updates the project in the Firebase RTDB.

Request Body: A JSON object containing the project ID and any necessary parameters for the Gemini API.


Example Request Body:
{
    "pid": "nataraj-12345",
    "parameters": {
        "style": "Ballet",
        "duration": 60
    }
}

Response: A JSON object indicating success and the generated dance steps.


Example Response:
{
    "success": true,
    "dance_steps": ["step1", "step2", "step3"]
}
